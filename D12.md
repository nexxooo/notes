# D12
## Le modèle entité-association
on ne se soucie pas de :
* des traitements et procédure
* de la facon dont elle seront stockées

**entité** = object ou individu

chaque objet possède un identifiant unique

une association entre objet est un lien --> pas d'existence propre 
* Une occurence d'association est un n-uplet d'occurence d'entité
* n est le nombre d'arcs de l'association : il est appelé **dimension** de l'association

### Role d'une entité
* Le role d'une entité dans une association définit la fonction qu'assure l'entité dans cette association
* Le role n'est précisé que si la clarification est nécessaire
* chaque entité donne naissance a une table 

### Règle de normalisation
* la meme propriété ne peut pas figurer dans plusieur entités 
* les propriété doivent avoir une raison d'être pour chasue occurence d'entité 
* une propriété ne peut pas avoir plusieur valeurs pour une meme occurence d'entité 
* une association de type 1,1/1,n ne peut pas être porteuse d'attributspo
